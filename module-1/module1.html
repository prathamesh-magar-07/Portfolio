Day 5

Initialize a Git repository: In your website's directory, run git init to initialize
a new Git repository. This will set up Git to track changes in your project.
Add files: Use git add . to stage all files in your directory, or specify individual files with git add filename.
Commit changes: Once you've staged the changes you want to include in the next commit, use git commit -m
"Your commit message" to commit them to the repository. Be sure to write a
descriptive commit message that explains the changes you've made. Create a repository on GitHub: Go to GitHub and
create a new repository. You can either create an empty repository or initialize it with a README file.
Set the remote repository: On your local machine, link your local repository to the GitHub repository you
just created by running git remote add origin repository URL. Replace repository
URL with the URL of your GitHub repository. Push changes to GitHub: Finally, use git push -u origin master
to push your committed changes to the master branch of your GitHub repository.
If you're working on a different branch, replace master with the name of your branch.
